# clawso-skins

Open authoring contracts and samples for **Clawso skins** and **Clawso pet packs**.

This repo is the creator authoring kit for Clawso. It contains generated schemas, human-readable specs, sample projects, the `skin-kit` CLI, and skill entrypoints that an agent can use to produce Creator Studio-compatible artifacts. The Clawso desktop client and marketplace remain the distribution and review surfaces.

> **Visibility gate.** Changing this repository to public is a GitHub settings action, not a code change. Before flipping visibility, confirm docs, examples, schema freshness, package metadata, and secret scans from the current default branch.

## Current Support

| Asset type | Authoring | Local validation | Local preview/build | Marketplace submission |
|------------|-----------|------------------|---------------------|------------------------|
| **Skin** | `skin.json`, token files, optional `skin.css` and bundled assets | CLI schema + security rules | CLI static shell preview and `.skin` bundle | Creator Studio, or CLI with certified creator token |
| **Pet pack** | `pet.json` vector pet-pack or designer-authored Rive manifest | CLI schema + bundled asset reference checks | Creator Studio runtime preview | Creator Studio |
| **Automations / Tools / Agents** | Not opened in this repo | Not opened | Not opened | Not opened |

## Two Authoring Paths

### Creator Studio

Use the Clawso client when you want a simplified, no-code flow:

1. Open **Developer / Creator Studio** in Clawso.
2. Choose **Create Skin** or **Create Pet**. Skin and pet authoring are separate lanes.
3. Import or edit the manifest/artifact generated by a designer, developer, or agent.
4. Preview locally: skins can be applied to the client preview; pet packs can run through the existing companion runtime.
5. Run BFF preflight with your signed-in certified creator identity.
6. Submit. A submission creates an immutable review snapshot.

Designers can use Creator Studio without installing this repo or running a CLI.

### CLI And Agent Automation

Use the repo-local CLI and skills when you want automation:

```bash
npm --prefix packages/skin-kit run build
node packages/skin-kit/dist/cli.js validate examples/default
node packages/skin-kit/dist/cli.js validate examples/pet-packs/pawsnap-puppy
node packages/skin-kit/dist/cli.js build examples/default
node packages/skin-kit/dist/cli.js publish examples/default --dry-run --api http://127.0.0.1:18808 --token test
```

Agents should load one of the skill entrypoints:

- [`skills/clawso-skin-creator/SKILL.md`](skills/clawso-skin-creator/SKILL.md)
- [`skills/clawso-pet-creator/SKILL.md`](skills/clawso-pet-creator/SKILL.md)

The skills may bootstrap the local CLI when it is available, but the final artifact should still be importable into Creator Studio. BFF preflight failures must be shown to the author; agents must not hide or "greenwash" failed policy checks.

## What Skins Can Change (v1)

Skins sit on a capability ladder. v1 exposes the first three rungs; arbitrary element repositioning and code execution are out of scope.

| Tier | You can change | Status |
|------|----------------|--------|
| **T1 Tokens** | colors, fonts, spacing, radius, shadow, motion | v1 |
| **T2 Appearance** | scoped CSS, bundled fonts/backgrounds/assets, glow/glass/animation | v1 |
| **T3 Layout (bounded)** | sidebar side, nav item order/visibility, top-bar widget order | v1 |
| **T4 Behaviour** | custom JS, new components, runtime plugins | not skin v1 |

A skin is a diff over the default skin: override only what you need, everything else inherits.

## What Pet Packs Can Change (v1)

Pet packs are one asset class, `pet-pack`. v1 supports declarative manifests, vector pet data, trigger-slot mappings, persona phrases, reactive FX, and bundled preview assets. Rive and richer bundled media are designer-authored inputs; use Creator Studio for runtime preview, BFF preflight, and submission.

Pet authoring is intentionally separate from skin authoring:

- Skin artifacts theme the client shell.
- Pet pack artifacts install into the companion runtime.
- Both can be generated by agents, edited by humans, previewed in Creator Studio, preflighted through Clawso BFF, and submitted to creator review.

## Publishing Model

- Creator authentication is required for BFF preflight and submission.
- Only certified creators can submit to review.
- Submitted snapshots are immutable; changing the work requires a new draft/submission.
- Rejected submissions are terminal for that attempt. Create a new attempt after fixing the artifact.
- Admin review can approve, reject, or spam-reject. Clawso does not run a heavy revision loop in v1.
- Published upgrades create new versions. Installed users keep their installed version until they choose to update.

## Generated Schema

Files under [`schema/`](schema/) are generated from the Clawso monorepo package `packages/skin-contract`. Do not hand-edit generated schema files. Regenerate them from the source contract and commit the generated output here.
